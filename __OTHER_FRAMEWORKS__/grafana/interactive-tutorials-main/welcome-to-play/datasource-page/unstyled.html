Welcome to Grafana's data source showcase! Data sources are the foundation of any Grafana dashboard - they connect your visualizations to the data that powers your insights. Grafana's extensive plugin ecosystem means you can connect to virtually any data source imaginable.

From traditional databases and time-series stores to modern APIs, cloud services, and even spreadsheets - if your data exists somewhere, Grafana can visualize it.

üåê Universal Data Access with Infinity

The Infinity plugin is one of Grafana's most versatile data sources, enabling connections to virtually any API or data format.

  • The Infinity data source plugin is a powerful and flexible connector that breaks down data silos. It supports multiple data formats including JSON for REST APIs, CSV for tabular data, XML for legacy systems, GraphQL for modern APIs, and general API endpoints. Infinity can fetch data from any HTTP endpoint, parse complex nested structures using JSONPath or UQL (Infinity Query Language), apply transformations, and even combine data from multiple sources. It's perfect for integrating third-party APIs, pulling data from custom applications, monitoring external services, or connecting to data sources without native Grafana plugins. The plugin includes advanced features like authentication (Bearer tokens, API keys, OAuth), custom headers, URL parameters, and response parsing options. Explore the Infinity data source plugin for universal API and data format connectivity.

üìã Log Analysis with Loki

Loki brings efficient log aggregation and querying to Grafana, making it easy to search and analyze logs alongside your metrics.

  • The Loki data source provides powerful log aggregation and querying capabilities designed to work seamlessly with Grafana. Unlike traditional log systems that index the full text of logs (which can be expensive), Loki only indexes metadata labels, making it cost-effective and highly performant. This example demonstrates parsing JSON logs from NGINX in a service mesh environment, showing how LogQL (Loki's query language) can filter, parse, and extract structured data from logs. You can search logs by labels, use regex patterns, calculate metrics from log data, correlate logs with metrics in the same dashboard, and create alerts based on log patterns. Loki excels at cloud-native logging, especially in Kubernetes environments, and integrates perfectly with Prometheus for complete observability. Discover Loki for efficient log aggregation and analysis.

üìä Spreadsheet Integration with Google Sheets

Bring your business data from Google Sheets directly into Grafana dashboards, bridging the gap between spreadsheets and observability.

  • The Google Sheets data source enables you to visualize spreadsheet data directly in Grafana, making it perfect for business metrics, manual data entry, or bridging the gap between business teams using spreadsheets and technical teams using Grafana. You can query specific sheets and ranges, automatically refresh data as the spreadsheet updates, combine spreadsheet data with other data sources in the same dashboard, and use Grafana's transformations to reshape and enrich the data. This is particularly valuable for OKRs and KPIs tracked in sheets, budget and financial data, manual inventory or capacity tracking, survey results and feedback data, or any business process that starts in spreadsheets. The plugin supports Google Workspace authentication and can access both public and private spreadsheets. Check out Google Sheets integration for visualizing spreadsheet data.

Beyond these featured examples, Grafana supports an extensive ecosystem of data sources:

Time Series Databases

  • Prometheus - The de facto standard for metrics and monitoring
  • InfluxDB - Popular time-series database for IoT and real-time analytics
  • Graphite - One of the original monitoring time-series databases
  • TimescaleDB - PostgreSQL extension for time-series data

Traditional Databases

  • PostgreSQL - Powerful relational database with rich querying
  • MySQL/MariaDB - Widely-used relational databases
  • Microsoft SQL Server - Enterprise database system
  • Oracle Database - Enterprise-grade database platform

Cloud & Distributed Systems

  • Elasticsearch - Search and analytics engine for logs and metrics
  • CloudWatch - AWS monitoring and observability service
  • Azure Monitor - Microsoft Azure's monitoring solution
  • Google Cloud Monitoring - GCP's observability platform

Application Performance

  • Jaeger - Distributed tracing system
  • Tempo - Grafana's distributed tracing backend
  • Zipkin - Distributed tracing for microservices
  • Pyroscope - Continuous profiling platform

Specialized & APIs

  • GitHub - Repository metrics and project tracking
  • JSON API - Generic JSON endpoint connector
  • CSV - Static or dynamic CSV file sources
  • TestData - Built-in data source for testing and demos

üéØ Choosing the Right Data Source

Selecting the appropriate data source depends on your data type and use case:

  • Real-time metrics? Prometheus, InfluxDB, or Graphite
  • Log aggregation? Loki, Elasticsearch, or CloudWatch Logs
  • Business data? Google Sheets, PostgreSQL, or MySQL
  • Custom APIs? Infinity plugin for maximum flexibility
  • Cloud services? CloudWatch, Azure Monitor, or GCP Monitoring
  • Distributed tracing? Jaeger, Tempo, or Zipkin
  • Application databases? PostgreSQL, MySQL, or SQL Server

üîó Combining Multiple Data Sources

One of Grafana's superpowers is the ability to combine data from multiple sources in a single dashboard:

  • Correlate metrics and logs - See Prometheus metrics alongside Loki logs
  • Business + technical metrics - Combine Google Sheets KPIs with application metrics
  • Multi-cloud visibility - Mix AWS CloudWatch, Azure Monitor, and on-prem data
  • Database + API data - Join SQL queries with REST API responses
  • Traces + metrics + logs - Complete observability in one place

üí° Best Practices

  • Use query variables - Make dashboards dynamic and reusable across environments
  • Optimize queries - Avoid pulling excessive data; use aggregations and filters
  • Set appropriate refresh rates - Balance freshness with system load
  • Secure your connections - Use authentication, encryption, and proper access controls
  • Document data sources - Add descriptions to help team members understand each source
  • Test mixed data source queries - Ensure data correlations make sense
  • Monitor data source health - Track query performance and availability

üîß Configuration Tips

  • Connection pooling - Configure appropriate timeouts and connection limits
  • Caching strategies - Use query caching for expensive or slow queries
  • Authentication methods - Choose between basic auth, API keys, OAuth, or service accounts
  • Data source provisioning - Use configuration-as-code for reproducible setups
  • Version compatibility - Keep data source plugins updated for bug fixes and features

üöÄ Advanced Techniques

  • Transformations - Reshape, filter, and combine data from any source
  • Query chaining - Use results from one query in another
  • Ad hoc filters - Enable dynamic filtering across dashboard panels
  • Mixed data sources - Create panels that query multiple sources simultaneously
  • Custom plugins - Build your own data source plugin for proprietary systems
